What can be the problem ?

Well actually to write a web page we can't just type text as we did. More to this text we also need to write what we call tags, those ones will give some instructions to the computer like "go to the next line" or "add an image"

The tags and there attributes

So this would be too easy, of course some engineer had to come along and complicate everything. It's not enough to just write text in the editor you also need to give instructions to the computer and this is why we have tags.

HTML pages are full of what we call tags, they are invisible on the screen for your visitors but they allow to the computer what he needs to show.

Those tags can be spotted easily. They are surrounded with chevron, which means < and > like the following: <tag>

What are there purpose exactly ?

They indicate the nature of the text they surround. They mean for instance "this is a title" "this is an image" etc

We distinguish two kinds  of tags the paired ones and the orphan one

  • The paired tags

    They open contain texts and are closed later on, here is an example:
    <title>This is a text</title>
    We distinguish the opening tag <title> and the closing one </title> that indicates that the title is finished, this tells the computer that everything that is not in this tage is not a title Exemple: this is not a title <title>this is a title</title> this is not a title

  • The orphan tags

    Those are the tags that serve usually to add an element at a specific place like an image.
    It is not necessary to state the begining and end of an image we will just say to the computer that we want an image here
    We use an orphan tag as follow:
    <image />

    Note that the last / is not mandatory but it is used to not misunderstand this tag to a paired tag, it's let say a convention among web developers

The attributes

The attributes are like options for a tag, they come to complete them and to give more informations to the computer, the attribute comes after the tag name and usually has a value: <tag attribute="value"...

What is it used for ?

Let's take our tag <image/> that we just saw, by itself it's not useful but we could add an attribute
that would indicate the name of the image to show: <image name="picture.jpg"/>
The computer then understands that he needs to show the image that is located in the file picture.jpg
In the case of a paired tag the attributes needs to be placed in the opening tag

Important some of the tags we have seen are fakes for you just to understand the concepts we will see the real ones right now !

The structure of an HTML5 page

Structure of an html5 page

Let's go back to our notepad and please type in there the same thing as there is on the picture below, which correspond to the basics of any HTML page:

There is some spaces at the begining of the lines to shift away some tags, this is not mandatory and won't change your page at the end but it makes the code easier to read, we call this indenting a page so you just need to press tab to do so in your text editor

You can see that the tags are opened and closed in a specific order.

For instance the <html> tag is the first to be opened and the last one to be closed (at the very end of the code with </html>).
The tags must be closed on the oposite way of there opening for instance:

  • <html><body></body></body> : this is correct a tag that is opened on the inside must be closed on the inside
  • <html><body></html></body>: this is incorrect the tags are one on top of the other
Can we have some details about the tags we have in our page ?

Of course let's explain the diferent ones

  • The Doctype

    <!DOCTYPE html>

    The first line is called the doctype.

    It is mandatory because that's the one that indicates that we are here creating an html page

    This is not really a tag like the others (it starts with an exclamation mark).

    We can say that its an exception that confirms a rule ;)

  • The <HTML> tag

    This is the main tag of the code, it frames up the whole code as you can see the closing tag is at the very end  of the code

  • The <head> and <body> tags

    A web page is composed of two parts:

    • The <head> : this section gives you some general information on the page like its title, the coding used for special characters etc… This section is generally short. The info it contains are directed to the computer but nontheless they are very important
    • The <body> : that's where is the main part of our page. Everything that we will write here will be displayed on screen. Inside of the body we will write the major part of the code.

    For now the body is empty (we will see that later). Let's focus on the two other tags in the head.

  • The encoding (charset)

    <meta charset="utf-8" />

    This tag indicates the kind of encoding used in your .html file

    Not going into much details here because it could get complicated really fast. The encoding indicates the way that the file is saved. It determines how the special characters will be displayed (accents, chinese symbols…)

    Even though there is a lot of diferent encoding we can use nowadays most of the developers use utf-8 it can display almost all possible characters and that's why we will use it here

  • The title of the page

    <title>

    This is the title of your page, maybe the most important element!

    Every page must have a title that describes what it contains, it's advised to use short titles less then a 100 characters

    The title won't be displayed on the page itself but on the tab as described in the picture below

Remember that the title will also appear in the results of search engines like google so choosing a good title is very important

The comments

Comments

We learnt how to create our first real html page in that chapter.

Before we finish i would like to present you the concept of the comments

A comment in html is a text that is used as a note or a remainder.

It is not displayed, it's not read by the computer it doesn't change anything to the page display

So it's not used for anything ?

Of course it is !

It is useful to you and to the persons that will read the source code of your page.

You can leave some comments to have some indications on how your page works

What's the point ?

It will help you remember how your page is working if you come back to your code after a long time. Don't laugh it happens to every webmaster.

Inserting a comment

A comment is a special html tag:

<!-- this is a comment -->

You can put it anywhere you want in your code there is no impact in the way the page is displayed but you can use it for instance to remember where you are in your code especially if it is a long code !

You can easily test this see this in every webpage because the browser has to dispaly the html every html page is public.

End of the story, everybody can see your html code and there is nothing you can do to stop them, so don't put there important information like passwords in comments ;)

So create a clean code easy to understand ! For you and others !

Summing up

Let's sum up what we have seen so far:

  • We use notepad to create a file that ends up in .html that will be our webpage
  • This file can easily be opened by any browser just clicking on it
  • In this file we will write the content of our page using html tags
  • The tags can have different types:
  • <tag> </tag> : they open and close to frame a content (begining and end)
  • <tag /> orphan tags (they don't have closing tag) and they are used to add something at a very specific place like an image
  • The tags can have attributes <image name="picture.jpg">
  • A webpage is composed of two sections <head> and <body>
  • We can see the source code of every webpage online.
  • We saw the very basic tags needed like <hX> the title tag where X goes from 1 to 6 and the paragraph tag <p>
